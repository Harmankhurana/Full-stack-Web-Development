What is API??
- A technology that powers communication between software on the internet.
- It's Application Programming Interface.
- It's essentially a set of rules and protocols that define how different software interact with each other.
- As we hit different routes using GET, POST, PUT, PATCH, and DELETE same way we can hit the API URL's with these routes methods,
to get the desired output

API endpoints -
BaseURL/Endpoint
ex - https://bored-api.appbrewery.com/random
here, https://bored-api.appbrewery.com/ is the Base URL
and, /random is the endpoint

Query Parameters -
After the endpoint I can put a key-value pair
ex - ?Query=value
for multiparams - ?query=value&query2=value
ex - https://bored-api.appbrewery.com/filter?type=charity
ex - https://bored-api.appbrewery.com/filter?type=education
ex - For multi params https://bored-api.appbrewery.com/filter?type=social&participants=2

What is JSON??
- JavaScript Object Notation is way to format data that can be sent over the internet in the readable but also efficient way.
- In JSON every key is a string and the values can be anything(Strings, numbers, arrays, etc.)

What is API Authentication??
Authentication verify the identity of a client attempting to access an API.

4 tiers of API Authentication -
 - No Authentication (Rate limit)
 - Basic Authentication (Username and Password)
 - API key Authorisation (uses API Key)
 - Token based Authentication (Generating Token with Username and Password)

use this https://secrets-api.appbrewery.com/ for practice

What is API Key??
- A unique identifier used to authenticate and authorize applications or users when they interact with an API

Syntax of different Routes(GET, PUT, PATCH, DELETE) in axios async await - 
1. GET -

app.get("/" , async (req , res) => {
  try {
    const result = await axios.get(URL , config);
    res.render("fileName" , { content : JSON.stringify(result.data)});
  } catch (error) {
    res.status(404).send(error.message);
  }
});

In "GET" route we have the URL and config object

2. POST -

app.post("/" , async(req , res) =>{
 try {
  const result = await axios.post(URL , body , config);
  res.sendStatus(201);
 } catch (error){
   res.status(404).send(error.response.data);
 }
});

In "POST" route we have the URL, body/data and config object

3. PUT -

app.put("/" , async(req , res) =>{
 try {
  const result = await axios.put(URL , body , config);
  res.sendStatus(201);
 } catch (error){
   res.status(404).send(error.response.data);
 }
});

PUT provides all of the data that we want to update in our backend

4. PATCH -

app.patch("/" , async(req , res) =>{
 try {
  const result = await axios.patch(URL , body , config);
  res.sendStatus(201);
 } catch (error){
   res.status(404).send(error.response.data);
 }
});

PATCH request which means that you can provide any bit of the data you want to update 
and everything else in the resource stays as it is

5. DELETE -

app.delete("/" , async(req , res) =>{
 try {
  const result = await axios.delete(URL, config);
  res.sendStatus(201);
 } catch (error){
   res.status(404).send(error.response.data);
 }
});
